# Tomato Diagnostic Center

## Architecture Overview
This is a comprehensive tomato leaf disease classifier application consisting of:
1. **Backend**: FastAPI server hosting an ensemble of ResNet-50 and EfficientNet-B0 ONNX models (`master_api.py`).
2. **Frontend**: Vite + React single-page application focused on aesthetic user experience (`frontend/`).
3. **Deployment**: Vercel (frontend + backend via serverless Python function).

## Dataset
- **Source**: 5 classes of tomato leaf images from Google Drive zip files.
- **Classes** (alphabetical, matching `ImageFolder` ordering):
  1. `Tomato___Bacterial_spot`
  2. `Tomato___Early_blight`
  3. `Tomato___Late_blight`
  4. `Tomato___Septoria_leaf_spot`
  5. `Tomato___healthy`
- **Total images**: 13,829
- **Split**: 80/10/10 (train/val/test) using `random_split` with `seed=42`

## Model Training Details

### Shared Configuration (Both Models)
| Parameter | Value |
|---|---|
| Input size | 224×224 |
| Batch size | 32 |
| Optimizer | AdamW (lr=5.97e-4, weight_decay=1e-4) |
| Loss function | CrossEntropyLoss |
| LR Scheduler | StepLR (step_size=7, gamma=0.1) |
| Epochs | 15 |
| Seed for data split | 42 |
| Normalization | ImageNet standard: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] |
| Train augmentations | Resize(224), RandomHorizontalFlip, RandomRotation(15), ColorJitter(0.1, 0.1) |
| Val/Test transforms | Resize(224), ToTensor, Normalize |

### EfficientNet-B0
- **Backbone**: `efficientnet_b0` (ImageNet pre-trained, full fine-tune)
- **Classifier head**: `nn.Dropout(p=0.4)` → `nn.Linear(1280, 5)`
- **Best val accuracy**: **98.55%** (Epoch 6)
- **Training time**: ~19 minutes (Colab T4 GPU)
- **Bug fix applied**: TransformSubset wrapper ensures proper augmentation during training
- **ONNX export**: opset 18, with dynamic batch axes

### ResNet-50
- **Backbone**: `resnet50` (ImageNet pre-trained, full fine-tune)
- **FC head**: `nn.Dropout(p=0.4)` → `nn.Linear(2048, 5)`
- **Best val accuracy**: **98.63%** (Epoch 8)
- **Training time**: ~32 minutes (Colab T4 GPU)
- **ONNX export**: opset 18, with dynamic batch axes
- **Bug fix applied**: TransformSubset wrapper ensures proper augmentation during training

## Resolved Issues

### ✅ Training Augmentation Bug (Fixed)
The original training code had a critical bug where `val_data.dataset.transform = ...` mutated the shared dataset object, causing training data to **never receive augmentation**. Both models have been **retrained** using a `TransformSubset` wrapper that gives each subset (train/val/test) its own independent transform. Training scripts with the fix are in `_training_ref/`.

### ✅ Ensemble Logic Bug (Fixed)
The ensemble averaging in `master_api.py` now uses raw float probabilities (0-1) for accurate averaging via `raw_probs`, then converts to percentage for the response. This replaced the old logic that averaged percentage-form values from the `distribution` dict.

### ✅ Smart Entropy-Weighted Ensemble
The ensemble now uses an **entropy-weighted averaging** mechanism. Instead of naive equal weights, models with sharper (lower entropy/more confident) probability distributions receive a higher weight. This prevents a confused model from dragging down the prediction of a highly confident model (e.g., resolving the ResNet Septoria/Early Blight confusion).

### ✅ Out-of-Distribution (OOD) Detection
Added a `validate_leaf_image` function in `master_api.py` that analyzes color histograms (green/brown/dark/yellow ratios) and texture. If an uploaded image does not appear to be a plant/leaf, the frontend displays a warning banner to prevent hallucinated diagnoses.

### Confidence Threshold
`master_api.py` includes a `CONFIDENCE_THRESHOLD = 60.0` — predictions below this threshold are flagged with `is_confident: false` in the response.

### ✅ LLM Personalised Advisor (Groq / Llama)
A new `POST /advisor` endpoint sends diagnostic context (disease, confidence, severity, model agreement, optional user growing context) to **Llama 3.3 70B** via **Groq** for fast, personalised treatment advice. The response is structured markdown with Assessment, Immediate Actions, Treatment Plan, Prevention, and When to Seek Help sections. Falls back gracefully if `GROQ_API_KEY` is not set.

### ✅ Visual Similarity Index
A new `POST /similar` endpoint extracts 1280-d CNN embeddings from EfficientNet-B0's penultimate layer and performs cosine similarity search against a pre-computed gallery of reference images. Returns top-K most similar reference cases with thumbnails and labels. Requires running `similarity/build_index.py` on Colab to generate the index.

## Backend (`master_api.py`)

### API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/` | Health check. Returns status, loaded models, class count. |
| GET | `/models` | Model metadata (name, params, val accuracy, ONNX size). |
| POST | `/predict?mode=<mode>` | Predict disease from uploaded image. |
| POST | `/heatmap?model=<mode>` | Generates an occlusion-sensitivity heatmap (Base64 PNG overlay). |
| POST | `/similar?top_k=3` | Find top-K visually similar reference cases from the gallery. |
| POST | `/advisor?disease_class=...&confidence=...` | Generate personalised LLM treatment advice via Groq/Llama. |

### Prediction Modes
- `ensemble` (default): Averages softmax probabilities from all loaded models, picks highest.
- `efnet`: Uses only EfficientNet-B0.
- `resnet`: Uses only ResNet-50.

### Image Preprocessing Pipeline (`preprocess_image()`)
1. Open image as RGB (handles alpha channels)
2. Resize to 224×224 with bilinear interpolation
3. Convert to float32, divide by 255.0
4. Normalize with ImageNet mean/std
5. Transpose HWC → CHW
6. Add batch dimension → `[1, 3, 224, 224]`

### Disease Knowledge Base (`DISEASE_INFO`)
Contains 5 entries, each with:
- `label`: Human-readable disease name
- `severity`: None / Moderate / Moderate-High / Critical
- `symptoms`: List of symptom descriptions
- `treatment`: List of treatment recommendations
- `prevention`: List of prevention strategies
- `precautions`: List of safety precautions

### Prediction Response Format
```json
{
  "success": true,
  "prediction": "Human-readable label",
  "prediction_class": "Tomato___class_name",
  "confidence": 95.23,
  "severity": "Moderate",
  "symptoms": ["..."],
  "treatment": ["..."],
  "prevention": ["..."],
  "precautions": ["..."],
  "models_agree": true,
  "mode": "ensemble",
  "diagnostics": {
    "efnet": {"prediction": "...", "confidence": 96.5, "distribution": {...}},
    "resnet": {"prediction": "...", "confidence": 94.0, "distribution": {...}}
  }
}
```

## Deployment (Vercel)

### Configuration (`vercel.json`)
- **Backend**: `api/index.py` → Vercel Python serverless function (max 250MB lambda)
- **Frontend**: `frontend/package.json` → `@vercel/static-build`, dist dir: `dist`
- **Routes**:
  - `/api/*` → Python function
  - `/predict` → Python function
  - `/models` → Python function
  - `/*` → Static frontend files

### Important Notes
- The ONNX model files (especially ResNet at ~94MB) are large. Vercel lambda size limit is 250MB.
- `onnxruntime` CPUExecutionProvider is used (no GPU in serverless).
- Model loading happens at startup via lifespan handler.
- **Environment Variables** (Vercel Dashboard):
  - `GROQ_API_KEY` — Required for the `/advisor` LLM endpoint
  - `GROQ_MODEL` — Optional, defaults to `llama-3.3-70b-versatile`
  - `VITE_API_URL` — Frontend API URL (set in Vercel for the frontend build)

## Key Instructions for Agents

### Backend Rules
- When modifying the backend, ensure the `DISEASE_INFO` dictionary reflects all fields expected by the frontend (`symptoms`, `treatment`, `prevention`, `precautions`).
- When dealing with ONNX models, note that `onnxruntime` CPUExecutionProvider is used due to the typical limits on deployment environments.
- Both models use **identical** preprocessing: 224×224, ImageNet normalization, CHW format.
- The ensemble averages softmax probabilities across models, then picks the argmax.

### Frontend Rules
- The UI uses a **clean, premium Light Theme** defined in `App.css` (featuring soft shadows, clean borders, and generous whitespace).
- Do NOT use Tailwind in the frontend; all styling is vanilla CSS.
- The app uses Outfit for display/headings and Inter for body text.
- All interactive states (hover, focus, active) should include smooth transitions.
- The app connects to the backend via `import.meta.env.VITE_API_URL` (with fallback to localhost).
- Keep the responsive breakpoint at 1100px for the main grid layout.
- The AI Advisor tab uses a purple/indigo gradient accent to distinguish LLM-powered features from the core green theme.
- Similar Cases cards use green borders for matching predictions and amber for differing ones.

### Model Retraining Rules
- Both models were retrained on Google Colab with T4 GPU using the **fixed** `TransformSubset` wrapper.
- Both use `random_split` with `seed=42` for reproducible train/val/test splits.
- The `TransformSubset` wrapper ensures train/val/test each get their own transforms without mutating the shared dataset.
- ONNX export uses `opset_version=18` with dynamic batch axes.
- Class mappings are generated from `datasets.ImageFolder` which sorts class directories alphabetically.
- **ResNet-50 Specifics:** To resolve class confusion (Septoria vs. Early Blight), ResNet-50 was retrained with advanced techniques: **Mixup** (alpha=0.2), **Label Smoothing** (0.1), **CosineAnnealingWarmRestarts**, and stronger spatial augmentations (RandomAffine, RandomPerspective, RandomErasing). Trained for 25 epochs, achieving ~97.6% validation accuracy.

### Training Scripts (Reference)
- Fixed training scripts are in `_training_ref/`:
  - `train_efficientnet_b0.py` — EfficientNet-B0 training script (with TransformSubset fix)
  - `train_resnet50.py` — ResNet-50 training script (with TransformSubset fix, Mixup, and Label Smoothing)
